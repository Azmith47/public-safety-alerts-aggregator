# Client Clarifications – Jason Howarth

Date: 16 March 2026

## Alert Storage
Store active alerts and retain historical alerts.
Users must be able to filter active vs expired alerts.

## Retention
Prototype may retain alerts for a limited window (e.g. 90 days).

## Subscriptions
Users save a filter (region, category, severity) and provide email.

Session 1 may implement a mock email notification system.

## Source Availability
System must display last successful refresh per source.

## Sources
Prototype should aggregate alerts from 2–3 credible sources
(Bureau of Meteorology, NSW RFS, SES suggested).

## Geography
Focus on NSW but schema should allow Australia-wide expansion.

## Alert Updates
Agency updates should update existing alerts, not create duplicates.

## Non-Software Context
Not required for this project.
